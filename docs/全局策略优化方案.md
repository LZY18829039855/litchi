# 全局策略优化方案

## 背景

当前客户端已经能通过一系列规则修复完成交付，但策略整体仍偏局部反应式：先按固定优先级处理任务、资源、移动、守卫和强制交付，再针对日志中暴露的问题补充阈值或特殊节点逻辑。

最近日志说明，单纯提前进入强制交付、跳过部分资源、优化 `S09 -> S10` 和 `S14` 验核可以带来收益，但这种方式容易继续堆单局参数。后续应把地图、天气、资源、任务、敌方位置和敌方分数纳入统一评估，让每一步动作都来自全局加权计算。

## 当前策略问题

- 路线选择主要围绕 `S14/S15`，遇到守卫、障碍、固定处理后再局部处理，缺少对全图多路线的持续比较。
- 强制交付依赖固定阈值，例如任务分和回合数，难以适配不同地图、不同水路优势和不同敌方节奏。
- 任务、资源、移动、设卡反制分散在不同优先级分支中，缺少统一的收益成本比较。
- 当前地图水路优势较大时，策略没有显式把水路路线作为高收益候选路线评估。
- 敌方分数高或交付进度领先时，缺少主动反制，例如在敌方必经瓶颈设置守卫或制造障碍。
- 对关键节点的处理仍有硬编码倾向，应改成动态识别当前地图上的关键瓶颈。

## 总体目标

将策略升级为“全局局面评估 + 候选计划打分 + 滚动重规划”。

每一回合先构建当前局面快照，生成多个候选计划，估算每个计划的最终收益和风险，选择分数最高计划的第一步执行。下一回合根据新的服务器状态重新规划。

这种模式可以避免一次性写死长计划，也能在敌方设卡、任务被抢、资源变化、天气变化或地图路线变化时及时调整。

## 规划流程

1. 构建 `WorldSnapshot`

   收集当前玩家、敌方玩家、全图节点、路线边、天气、资源、任务、守卫、障碍、已处理节点、当前阶段、剩余回合等信息。

2. 估算全局路线

   分别计算当前位置到 `S14 -> S15` 的多条可行路线，包括陆路、水路、山路、绕路和含资源路线。路线成本应包含移动距离、天气影响、固定处理耗时、守卫风险、障碍风险和窗口争夺风险。

3. 生成候选计划

   候选计划不只是下一步动作，而是短期意图，例如直接赶路、走水路、顺路做任务、绕路拿资源、设卡拖敌方、破卡、清障、强制通行、到 `S14` 绑定 `BREAK_ORDER` 验核等。

4. 统一打分

   使用同一套评分函数比较所有候选计划，避免任务、资源、赶路和反制互相割裂。

5. 执行第一步

   只提交当前回合法律动作。下一回合重新构建快照并重新选择。

## 评分模型

建议先使用可解释的线性加权模型：

```text
plan_score =
  delivery_score_estimate
+ task_gain
+ resource_gain
+ enemy_delay_gain
+ route_advantage_gain
- time_cost
- freshness_cost
- good_fruit_cost
- illegal_risk
- contest_risk
- deadlock_risk
```

关键指标：

- `time_cost`：执行该计划后预计到达 `S15` 的剩余回合数。
- `task_gain`：任务分收益，扣除绕路和处理耗时。
- `resource_gain`：资源预计节省的时间、保鲜收益或终局收益。
- `enemy_delay_gain`：设卡、障碍、争夺等对敌方可能造成的延迟。
- `route_advantage_gain`：水路、捷径或天气有利路线的额外收益。
- `freshness_cost`：移动、天气、疾行等造成的鲜度损失。
- `good_fruit_cost`：破卡、强行、急策等消耗好果带来的分数损失。
- `illegal_risk`：动作被拒绝或状态不满足的风险。
- `contest_risk`：资源、任务、固定处理或宫门验核争夺风险。
- `deadlock_risk`：长时间等待守卫、冷却或窗口失败导致卡住的风险。

## 地图非固定处理

不应把具体节点作为核心策略写死。可以动态识别瓶颈：

- 计算多条高分路线。
- 找出多条路线共同经过的节点或边，作为当前地图的关键瓶颈。
- 如果敌方靠近瓶颈或已经设卡，提高绕路、破卡、强制通行、提前用马的权重。
- 如果我方领先且敌方预计必经瓶颈，提高设卡或制造障碍的权重。
- 如果水路路线预计时间更低，自动提高水路节点和水路资源的优先级。

这样 `S09/S10` 可以退化为当前地图中的一个瓶颈案例，而不是特殊规则中心。

## 任务策略

任务选择应看边际收益，而不是只看固定任务分阈值。

- 顺路任务：如果任务分除以额外耗时高于阈值，就做。
- 高分任务：允许适度绕路，但必须保证预计能安全交付。
- 低分任务：只有不影响主路线、敌方不明显领先时才做。
- 我方任务分落后：提高任务权重，尤其是高分和顺路任务。
- 我方任务分已高：降低任务权重，提高交付时间权重。
- 冲刺阶段：默认不做普通任务，只保留清障型任务等同时提升通行和得分的动作。

当前类似 `taskScore >= 60` 后提前赶路的逻辑，应改为：

```text
如果继续做任务的预计净收益 < 立即赶路的时间收益，则赶路。
```

## 资源策略

资源应按预计实际收益选择：

- 马类资源：优先用于后续最长、最耗时或最关键的边，而不是固定某条边。
- 船权和水路相关资源：当前地图水路优势明显时，提高领取和争夺权重。
- 冰鉴：只在鲜度影响最终分或交付安全时使用，避免尾段浪费动作。
- 过所、官凭、破关相关资源：按能减少的固定处理、窗口、验核或通行成本估值。
- 情报和小分队探路：如果能减少高成本固定处理或关键验核时间，纳入收益。
- 资源争夺：高价值资源可争，低价值资源在敌方接近时放弃。

## 敌方策略

每回合应推断敌方状态：

- 敌方预计交付时间。
- 敌方任务分和综合领先情况。
- 敌方当前位置到 `S14/S15` 的高分路线。
- 敌方是否会经过我方可设卡节点。
- 敌方是否正在争同一任务、资源或固定处理。

动态调整方向：

- 敌方时间领先但任务分低：优先提速交付。
- 敌方任务分高且交付不慢：提高补任务分权重。
- 敌方即将经过瓶颈：如果不显著拖慢自己，考虑设卡或制造障碍。
- 敌方争同一资源或任务：按净收益决定争夺或绕开。
- 敌方接近 `S14`：提高终局反制和自身快速验核交付权重。

## 候选计划示例

候选计划可以先覆盖以下类型：

- `DIRECT_DELIVERY`：直接向 `S14/S15` 前进。
- `WATER_ROUTE`：选择当前水路优势路线。
- `TASK_DETOUR`：绕路或顺路完成一个任务。
- `RESOURCE_DETOUR`：绕路领取高收益资源。
- `CHOKE_GUARD`：在敌方预计必经瓶颈设卡。
- `CLEAR_OR_T04`：通过清障或 T04 任务处理障碍。
- `BREAK_OR_FORCE_PASS`：破敌方守卫或强制通行。
- `RUSH_GATE`：在 `S14` 使用 `VERIFY_GATE + BREAK_ORDER`。

每个计划至少输出：

```text
type
first_action
eta_to_delivery
expected_score_delta
self_delay
enemy_delay
risk
total_score
reason
```

## 日志与可观测性

策略日志应从“执行了什么”升级为“为什么执行”：

```text
PLAN direct_delivery eta=516 score=120 risk=5
PLAN water_route eta=492 score=138 risk=8
PLAN task T_005 extra=4 gain=15 score=126 risk=6
PLAN set_guard target=S10 enemyDelay=18 selfDelay=1 score=142 risk=10
CHOOSE set_guard first_action=SET_GUARD reason=enemy leading and must pass choke
```

这样后续失败时，可以快速判断问题来自路线估值、敌方预测、动作执行、规则理解还是参数权重。

## 建议落地顺序

第一阶段：新增全局评估模块

- 新增 `planner.py` 或类似模块。
- 实现 `WorldSnapshot`、路线估值、敌方预计到达时间、任务净收益、资源净收益。
- 先把 `_should_force_delivery` 改为调用全局评估结果。

第二阶段：替换任务和资源选择

- 将 `_handle_tasks` 和 `_handle_resources` 的固定优先级改为候选打分。
- 支持当前节点、邻近节点、顺路节点的任务和资源比较。
- 保留现有动作合法性检查，避免引入非法动作。

第三阶段：加入敌方反制

- 实现敌方路线预测。
- 评估设卡、障碍、争夺、绕路和直接赶路的净收益。
- 只在敌方领先或反制收益显著时触发。

第四阶段：扩大验证

- 使用多份历史日志构造烟测场景。
- 覆盖水路优势、敌方任务分高、敌方交付快、关键瓶颈被守、资源争夺、任务绕路等情况。
- 每次策略选择都输出计划评分，便于人工检查。

## 验收标准

- 不依赖固定地图节点编号即可选择水路、陆路和绕路。
- 强制交付不再只由固定回合数触发，而由预计净收益决定。
- 敌方领先时能主动评估设卡、障碍或争夺。
- 任务分高价值时能继续做任务，任务净收益不足时能及时赶路。
- 资源领取由实际节省时间和风险决定。
- 日志能解释每次动作选择原因。
